EIK FREEZE NOTE

Release:
Shunyaya Execution Identity Kernel (EIK) v1

Purpose:
This release freezes the minimal public demonstration boundary for EIK.

Core claim:
EIK demonstrates deterministic execution identity.

The governing behavior is:

same observable boundary -> same execution certificate
changed observable boundary -> changed execution certificate
restored observable boundary -> restored execution certificate

Execution identity model:

B = (target_sha256, argv, stdin_text, stdout_text, stderr_text, exit_code)

E = SHA256(JSON_canonical(B))

Where:

B represents the observable execution boundary.
E represents the deterministic execution certificate produced by the kernel.

Meaning of freeze:

Any modification inside the frozen boundary requires:

regeneration of outputs_example

re-running verification

confirming replay and mutation behavior

updating this freeze note if the boundary definition changes

Scope of certification:

EIK certifies structural execution identity only.

EIK does not certify:

algorithm correctness

domain validity

safety

optimization quality

performance superiority

full hermetic environment reproducibility

This freeze preserves a minimal, replay-verifiable public demonstration of deterministic execution identity.
